## Task Assignments

_Tasks are numbered sequentially within this phase using zero-padded two-digit IDs + a
descriptive slug (e.g. `task01_core_engine`). The slug makes folder and branch names
self-documenting. Sub-phase labels in PLANNING.md (e.g. 1a, 1b) are planning groupings only —
the task ID here is the authoritative collaboration ID used in folder, branch, and doc names._

**Naming convention:**
- Folder: `docs/version1/phases/phase[#]/task[NN]_[slug]/`
- Branch: `[username]/phase[#]_task[NN]_[slug]`
- Doc prefix: `RESEARCH_PHASE[#]_TASK[NN].md` (slug not repeated in doc name)
