# Migration: rpid 2.0.0 → 2.1.0

> Additive (MINOR) — mostly drop-in. Read by `prompts/UPGRADE_TEMPLATE_PROMPT.md`. Project content
> (`docs/version1/*`, `src/**`, your filled `CLAUDE.md` sections) is never touched.

## Update brief (what's new & why you'd want it)

GitHub triggers that make the autonomous loop real end to end: label an issue `claude` and Claude
handles it (reproduce → fix → test → PR) and opens a pull request you can review and iterate on with
`@claude` comments. The feedback loop.

## Drop-in (new files — just add)

- `.github/workflows/claude-issue-to-pr.yml` — the autonomous issue → PR workflow.

## Replace (template-owned — swap if unmodified; if customized, reconcile)

- `.github/workflows/claude.yml` — permissions bumped read → write so `@claude` mentions can push
  changes to a PR (not just comment). The action still only acts on mentions from users with write
  access, so this stays safe.
- `CLAUDE.md` — the workflow list in the Track 3 / CI-test-gate note now includes
  `claude-issue-to-pr.yml`.
- `prompts/modes/ISSUE_MODE.md` — the Trigger Condition note now points to the new workflow + the
  `claude` label.

## Manual steps (one-time, on GitHub)

1. Add the repo secret **`CLAUDE_CODE_OAUTH_TOKEN`** (`claude setup-token`, or the GitHub App
   install flow) if it isn't already set.
2. Create a **`claude`** label (Issues → Labels). Adding it to an issue triggers a run.
3. (Optional) To run on **every** new issue instead of the label opt-in, swap the `on:` / `if:`
   lines in `claude-issue-to-pr.yml` per its inline comments.

## Verify

- Label a throwaway issue `claude`; the "Claude Issue → PR" workflow runs and opens a PR that closes
  it.
