# Migration: rpid 1.x → 2.0.0

> Read by `prompts/UPGRADE_TEMPLATE_PROMPT.md`. Its first job is to **explain the release**, then
> apply it by ownership (`.rpid/ownership.md`). Project content (`docs/version1/*`, `src/**`, your
> filled `CLAUDE.md` sections) is never touched.

## Update brief (what's new & why you'd want it)

v2.0 fuses Garry Tan's GStack methodology into the RPID loop. You gain:
- **Role-based reviews** at each gate — CEO scope, Eng architecture, Code pre-merge audit (+ optional
  Security / Design / DevEx).
- **Ship + Reflect** stages after Track 3 — plan-completion audit, retro, persisted learnings.
- A **Review-autonomy dial** (`full` / `milestone` / `auto`) so you choose how hands-on to be, with
  a Hard-Gates safety floor that always stops for dangerous/irreversible things.
- A **test-gate bootstrap** (`TEST_SETUP`) that fills the failing `test.yml` placeholder — resolves
  ADR-002.
- This **versioning + self-link + upgrade** system.

## Breaking / semantic changes (need your input)

1. Track 4's rule is now named the **Iron Law** ("no fix without investigation"). Update local
   references.
2. `CLAUDE.md` methodology is now wrapped in `<!-- RPID:METHODOLOGY:* -->` fences. If your
   `CLAUDE.md` predates them, we add the fences around your Ethos/Collaboration region (your
   Project / North-Star content stays put).
3. Two new feature-flag rows (`Review autonomy` = `full`, `GStack role reviews` = ON) — same
   behavior as today unless you change them.
